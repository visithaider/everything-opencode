## Go Best Practices

### Code Style
- Follow Go naming conventions
- Use meaningful variable names
- Keep functions small (< 50 lines)
- Proper error handling (don't ignore errors)

### Error Handling
- Always handle errors
- Use custom error types
- Wrap errors with context
- Don't use panic in production

### Concurrency
- Use goroutines for concurrent operations
- Use channels for communication
- Avoid race conditions
- Use sync package when needed

### Performance
- Use strings.Builder for string concatenation
- Pool resources with sync.Pool
- Avoid unnecessary allocations
- Use bufio for I/O

### Testing
- Table-driven tests preferred
- Use testify or standard testing
- Benchmark performance-critical code

## Go Specific Checks

- [ ] No unused imports
- [ ] No unused variables
- [ ] Proper error wrapping
- [ ] Context usage for cancellation
- [ ] Resource cleanup (defer)
- [ ] No mutex copies
- [ ] Proper slice capacity
- [ ] HTTP handler errors logged

## Commands

```bash
# Format code
go fmt ./...

# Lint
golangci-lint run

# Vet
go vet ./...

# Test
go test -v ./...

# Coverage
go test -coverprofile=coverage.out ./...
```
